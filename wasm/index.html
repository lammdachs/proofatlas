Research Focus

How can we best represent logical clauses as graphs and learn to select useful clauses during proof search?

Graph Representations

Clauses as syntax trees with typed nodes (clause, literal, predicate, function, variable, constant) and 13-dimensional features encoding structure and metadata.

Graph Neural Networks

Learning embeddings from clause structure using GCN, GAT, GraphSAGE, and other architectures to capture logical patterns.

Transformers/Attention

Using attention mechanisms over graph embeddings to score clauses based on their relevance to the proof goal.

Learned Selection

Replacing hand-crafted heuristics (age-weight ratio) with neural policies trained on successful proofs.

Technical Stack

  • Rust core: High-performance saturation prover with superposition calculus
  • Burn: Pure Rust ML framework for clause selection
  • PyTorch: Model training with safetensors export
  • WebAssembly: Client-side proving with ML selection

ProofAtlas runs entirely in your browser. No data is sent to any server.
